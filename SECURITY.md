# Security Policy

## Reporting a Vulnerability

Please do not open a public GitHub issue for security vulnerabilities.

Instead:

- use GitHub private vulnerability reporting if the repository has it enabled
- otherwise contact the repository owner privately through GitHub

Include:

- a clear description of the issue
- impact assessment
- reproduction steps
- a minimal proof of concept if possible
- any suggested remediation

## Response Expectations

The target process is:

- initial acknowledgment within 3 business days
- triage and severity assessment as soon as reasonably possible
- coordinated disclosure after a fix is available

## Scope

Report vulnerabilities related to:

- remote code execution
- arbitrary file access
- SSR injection paths
- unsafe compiler transforms
- sandbox escapes
- supply-chain exposure introduced by project tooling

If you are unsure whether something is security-sensitive, report it privately first.
