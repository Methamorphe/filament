# Licensing and IP

This repository is public and open source, but that does not transfer ownership of the code.

## Default License

Unless a file says otherwise, source code in this repository is licensed under the Mozilla Public License 2.0 (`MPL-2.0`).

See:

- [LICENSE](./LICENSE)
- [TRADEMARKS.md](./TRADEMARKS.md)

## Ownership

Copyright remains with the original author or contributor of each change.

For now, the project is intentionally keeping the contributor surface narrow so the chain of title stays simple and investor-ready.

## Contribution Policy

At this stage:

- issues, ideas, bug reports, and design feedback are welcome
- unsolicited code contributions may be declined
- maintainers may request additional IP paperwork before accepting non-trivial third-party code

The current practical default is maintainer-authored code only until the legal entity and contribution workflow are finalized.

## Future Commercial Options

This setup keeps open the option to:

- offer commercial licenses
- dual-license future releases
- keep proprietary modules or hosted products outside this repository

Important: code that has already been released under an open-source license cannot be retroactively made proprietary for recipients of that release.

## Trademark and Brand

Open-source licensing applies to the code.
It does not grant rights to use project names, logos, or branding in a way that suggests endorsement.

See [TRADEMARKS.md](./TRADEMARKS.md).

## Practical Position

If your immediate goal is:

- maximum adoption: keep the current license or move to Apache-2.0 later
- stronger reciprocity on framework changes: keep MPL-2.0
- maximum control over commercial use: use a source-available license instead, but that would no longer be open source

## Legal Note

This document is an operational policy note for the repository.
It is not legal advice and should be reviewed by counsel before fundraising or opening external code contributions broadly.
